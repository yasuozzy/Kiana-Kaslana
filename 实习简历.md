## 项目经历

### 切尔西官方商城 - 体育用品电商平台 ⭐️ 核心项目

**项目时间**：2024年9月 - 2024年10月（持续优化中）  
**项目简介**：一个基于 Vue 3 生态的现代化电商平台，模拟切尔西官方商店，实现了用户系统、商品展示、购物车、个人中心等完整业务流程  
**技术栈**：`Vue 3` + `Vuex` + `Vue Router` + `Element Plus` + `Arco Design` + `Axios` + `Vite` + `Less`  
**项目地址**：[GitHub链接（填写你的地址）] | [在线演示（可选）]

**项目规模**：

- **代码量**：5000+ 行（纯手写，无脚手架生成代码）
- **组件数量**：25+ 个业务组件 + 10+ 个公共组件
- **路由页面**：20+ 个页面，包含3级嵌套路由
- **状态管理**：使用 Vuex 模块化管理（cart + user 模块）
- **商品数据**：100+ 件商品，8 大分类，20+ 子分类

#### 核心功能与技术实现

**1. 用户认证系统**

- **技术方案**：基于 Vuex + localStorage 实现用户状态持久化
- **功能实现**：
  - 开发用户注册/登录表单，使用 Element Plus 进行表单验证
  - 通过 Promise 封装异步登录流程，实现邮箱唯一性校验
  - 实现登录状态持久化，解决页面刷新后用户信息丢失问题
  - 配置路由守卫，保护需要登录才能访问的页面
- **技术亮点**：登录状态同步到 localStorage，页面刷新后自动恢复用户信息

**2. 购物车系统** 🔥 技术难点

- **技术方案**：Vuex 模块化管理 + 前后端数据同步
- **功能实现**：
  - 实现添加/删除商品、修改数量、更新规格（尺寸/颜色）等 8 个核心操作
  - 处理相同商品不同规格的识别和数量合并逻辑（基于 id + size + color 三维判断）
  - 开发购物车弹窗组件，实现 Mutation + Action 的标准状态管理流程
  - 实现购物车价格自动计算，使用 reduce 方法聚合商品总价
  - 使用 localStorage 实现购物车数据持久化，与后端数据双向同步
- **技术难点**：
  - 解决购物车项ID（cartItemId）与商品ID（productId）的区分问题
  - 实现前端状态与后端数据的一致性维护
- **性能优化**：使用 getter 缓存计算属性，避免重复计算购物车总价

**3. 商品展示与筛选**

- **功能实现**：
  - 开发商品分类页，实现 6 大类 + 20+ 子分类的多级导航
  - 使用 v-for + 计算属性实现动态商品筛选（按分类、子分类、价格、颜色等）
  - 实现商品列表分页功能，支持 24/48/72 三种每页显示数量切换
  - 开发商品详情页，包含缩略图选择、主图展示、尺寸选择、库存提示等
  - 实现"最近浏览"功能，记录用户浏览的最新 10 件商品
- **交互优化**：
  - 筛选条件实时响应，使用计算属性优化筛选性能
  - 缩略图点击切换主图，提升用户体验

**4. 路由架构设计**

- **技术方案**：多级嵌套路由 + 路由懒加载 + 路由元信息
- **功能实现**：
  - 配置 15+ 个路由，包含父子嵌套关系（如 `/football-gear/home`）
  - 使用路由元信息（meta）传递分类信息，实现组件复用
  - 通过路由守卫（beforeEach）动态设置页面标题
  - 使用动态导入（`import()`）实现路由懒加载，优化首屏加载速度
- **技术亮点**：通过计算属性从 `$route.meta` 动态获取分类，避免 props 传递复杂度

**5. 接口对接与数据交互**

- **技术方案**：Axios 封装 + RESTful API
- **功能实现**：
  - 对接购物车相关接口（POST /cart/add、DELETE /cart/remove/:id、GET /cart/:userId 等）
  - 处理接口响应数据，映射到 Vuex 状态
  - 实现请求错误处理和用户提示
- **数据处理**：理解前端商品数据结构与后端返回数据的映射关系

**6. 组件化开发**

- **组件设计**：
  - 抽象公共组件：顶部导航栏、页脚、商品卡片、分页器等
  - 业务组件：购物车弹窗、商品筛选面板、商品详情等
  - 实现组件间通信：props 传递、自定义事件、Vuex 全局状态
- **代码复用**：同一个商品列表组件，通过路由参数支持不同分类的展示

**项目成果与收获**：

- ✅ **功能完整度**：实现了从首页浏览 → 商品筛选 → 加入购物车 → 用户登录 → 订单管理的完整业务闭环
- 📝 **代码质量**：代码注释率 ≥ 40%，遵循组件化、模块化开发规范，具备良好的可维护性
- 🚀 **技术成长**：从零到一完成 Vue 3 全家桶项目，掌握了现代前端工程化开发流程
- 💡 **问题解决**：独立解决了购物车 ID 冲突、数据持久化、组件通信等技术难点
- 🎯 **业务理解**：深入理解电商业务逻辑，具备前后端协作和接口对接经验
